# Два Голоса

Публичный адаптивный тест для пары с двумя независимыми режимами прохождения:

- **одно устройство** — первый участник отвечает и передаёт телефон второму;
- **разные устройства** — создаются две отдельные секретные ссылки, ответы синхронизируются через Supabase.

## Текущая версия

Анкета: **30 вопросов, 10 сфер, версия `2026-07-v3`**.

Сферы:

1. доверие;
2. общение;
3. конфликты;
4. поддержка;
5. близость;
6. границы;
7. ревность и безопасность;
8. деньги;
9. быт;
10. будущее.

Количество вопросов намеренно не увеличено по сравнению с v2. В v3 часть формулировок стала более конкретной и поведенческой, а чувствительные негативные утверждения обрабатываются в обратной шкале.

## Что выдаёт отчёт

После завершения обеих анкет формируются:

- индекс пары;
- отдельная согласованность взглядов;
- средняя оценка качества отношений;
- персональный архетип пары;
- 10 совместных сфер;
- 4 мета-системы отношений: безопасность, связь, координация и направление;
- индивидуальный профиль каждого участника;
- кто в среднем оценивает отношения теплее — с явным пояснением, что это не показатель «кто любит больше»;
- конкретный вопрос с максимальным расхождением ответов;
- конкретный вопрос с самым сильным совпадением;
- сфера, где партнёры сильнее всего живут в разных версиях отношений;
- три сильнейшие опоры;
- три главные зоны внимания;
- отдельный блок контрольных сигналов по вопросам давления, угроз, контроля и использования уязвимости;
- практический план на 7 дней;
- пять вопросов для совместного разговора;
- возможность поделиться кратким результатом;
- сохранение результата в PDF.

Основной индекс — авторская эвристика: **45% согласованность восприятия + 55% средняя оценка качества отношений**. Это не валидированный психологический тест и не прогноз будущего пары.

## Необязательные даты рождения и астро-бонус

В v3 можно указать даты рождения обоих партнёров.

- поля полностью необязательны;
- если хотя бы одна дата не указана, астрологический блок в результате не показывается вообще;
- даты рождения не участвуют в индексе пары, оценке сфер, контрольных сигналах или рекомендациях;
- при наличии обеих дат появляется отдельный блок **«Астро-бонус»** со знаками зодиака, стихиями и развлекательной интерпретацией сочетания;
- сам блок прямо помечен как развлекательный и не имеющий доказательной силы.

В сетевом режиме даты рождения временно хранятся вместе с записью пары и удаляются вместе с ней. Сессия пары имеет срок действия 30 дней.

## Интерфейс

Основная тема — светло-голубая и mobile-first:

- крупные зоны нажатия;
- на телефонах варианты ответа идут вертикально с текстовыми подписями;
- нижняя навигация во время вопросов закреплена и учитывает safe-area iPhone;
- отчёт перестраивается в одну колонку на узких экранах;
- панель действий с результатом остаётся доступной;
- карточки рассчитаны на длинный итоговый отчёт без перегрузки экрана.

## Структура

- `public/index.html` — главная разметка и маркетинговая подача;
- `public/styles.css` — светло-голубая адаптивная тема;
- `public/app.js` — вопросы, режимы прохождения, расчёты, отчёт, астро-бонус и sharing;
- `supabase/schema.sql` — таблица и защищённые RPC-функции сетевого режима;
- `.github/workflows/pages.yml` — публикация через GitHub Pages.

## Обязательная миграция Supabase после перехода на v3

После обновления кода нужно **один раз повторно выполнить весь актуальный `supabase/schema.sql` в Supabase SQL Editor**.

Миграция:

- сохраняет старые записи;
- добавляет необязательные `birth_a` и `birth_b`;
- переводит новые пары на версию `2026-07-v3`;
- оставляет 30 ответов для v2/v3 и 18 для старой v1;
- заменяет старую сигнатуру `create_couple_session`, чтобы PostgREST не видел неоднозначные RPC-функции;
- перезагружает schema cache PostgREST.

После миграции для проверки v3 нужно создавать **новую пару**. Старые ссылки предыдущих версий намеренно не открываются новой анкетой.

## Приватность и безопасность

- Publishable key Supabase является публичным клиентским ключом и находится в браузерном коде намеренно.
- `service_role` / Secret key в проекте не используется.
- Таблица `couple_sessions` имеет RLS, а прямые права `anon`/`authenticated` на таблицу отозваны.
- Браузер работает только через ограниченные `SECURITY DEFINER` RPC-функции.
- Для каждого партнёра генерируется отдельный случайный токен; в БД хранится только SHA-256-хэш токена.
- Ответы обоих участников возвращаются клиенту только после завершения обеих анкет.
- Даты рождения необязательны и не используются в основном анализе.
- Запись пары может быть удалена пользователем из отчёта.

## Ограничение

Проект не является психологической диагностикой, медицинским сервисом или инструментом прогнозирования расставания. Перед массовым коммерческим запуском потребуются rate limiting / CAPTCHA, юридические документы, политика конфиденциальности, мониторинг и формализованная политика хранения данных.
