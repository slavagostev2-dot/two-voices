# Два Голоса

Публичный адаптивный тест для пары с двумя режимами прохождения:

- **одно устройство** — первый участник отвечает и передаёт телефон второму;
- **разные устройства** — создаются две секретные ссылки, ответы синхронизируются через Supabase.

## Версия 2

Текущая анкета: **30 вопросов, 10 сфер, версия `2026-07-v2`**.

Сферы анализа:

1. доверие;
2. общение;
3. конфликты;
4. поддержка;
5. близость;
6. границы;
7. ревность и безопасность;
8. деньги;
9. быт;
10. будущее.

После завершения обеих анкет формируются:

- индекс пары;
- отдельная согласованность взглядов;
- средняя оценка качества отношений;
- 10 совместных сфер;
- индивидуальный профиль каждого участника;
- сильнейшая общая опора;
- главная зона внимания;
- сферы с наибольшей разницей восприятия;
- персональные вопросы для совместного разговора.

Формула индекса пары прозрачная: **55% совпадение восприятия + 45% средняя оценка качества отношений**. Это авторская эвристика продукта, а не валидированный психологический тест.

## Интерфейс

Основная тема — светло-голубая. Интерфейс сделан mobile-first:

- крупные зоны нажатия;
- вертикальная шкала ответов на узких экранах;
- закреплённая нижняя навигация во время вопросов;
- одноколоночные отчёты на телефонах;
- адаптивные карточки результатов;
- корректные safe-area отступы для мобильных устройств.

## Структура

- `public/index.html` — основная разметка;
- `public/styles.css` — светлая адаптивная тема;
- `public/app.js` — вопросы, режимы прохождения, расчёты и отчёт;
- `supabase/schema.sql` — таблица и защищённые RPC-функции сетевого режима;
- `.github/workflows/pages.yml` — публикация через GitHub Pages.

## Обязательное обновление Supabase после перехода с v1 на v2

Если раньше уже выполнялся `schema.sql` для 18 вопросов, после обновления до v2 нужно **один раз повторно выполнить весь текущий файл `supabase/schema.sql` в Supabase SQL Editor**.

Миграция сохраняет старые тестовые записи с 18 ответами на уровне таблицы, но новые сессии используют 30 ответов и версию `2026-07-v2`.

## Приватность и безопасность

- Publishable key Supabase является публичным клиентским ключом и находится в браузерном коде намеренно.
- `service_role` / Secret key в проекте не используется.
- Таблица `couple_sessions` имеет RLS, а прямые права `anon`/`authenticated` на таблицу отозваны.
- Браузер работает только через ограниченные `SECURITY DEFINER` RPC-функции.
- Для каждого партнёра генерируется отдельный случайный токен; в БД хранится только SHA-256-хэш токена.
- Ответы обоих участников возвращаются клиенту только после завершения обеих анкет.
- Сессия пары истекает через 30 дней.

## Ограничение

Проект не является психологической диагностикой. Перед массовым коммерческим запуском нужны антиспам/rate limiting, CAPTCHA, юридические документы, мониторинг и отдельная политика хранения данных.
